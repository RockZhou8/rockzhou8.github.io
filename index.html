发表于 2016-04-03   |   分类于 个人译文   |  

原文：
Accessible, Simple, Responsive Tables

你必须承认，没有一种单一地能使表格响应式表现的方法，然后我们再来想办法来解决。Davide Rizzo 说过的这个我很赞同，并且这一伟大的领域我们已经实践了一段时间。

内容和比较表

在一些有各种类型表格的网页上，内容会任意的疯狂地响应式的变化。我觉得最棘手的是对照表和普通内容布局表，而且也没有通用的基于 CSS 的解决方案。

我寻求一个灵活和简单的解决方案，一个可复用的 Web 组件，并且可以不管其中的内容。


对照表的例子

表标记还能为我所用吗？

标准表的标记似乎很有语义感，并且对其单元格也很有用。我主要关注的是通俗易懂。本地表标记能够使用户通过屏幕阅读器获取内容并定位吗？

我用一个简单的表格做了一些测试。通过一些屏幕阅读器（Chrome Vox 和 VoiceOver），我试图定位标记：

  • ChromeVox 会告诉你有个表格，VoiceOver 还会告诉你有多少行和列（这更有用）。
  • ChromeVox 或 VoiceOver都会告诉你表的标题。， 甚至 scope=”row|col” 似乎并没有什么用。只有一种识别他们的方法就是把它包进 标记中。
  • 阅读器逐行遍历表格，也并不会管表的内容如何。VoiceOver 至少可以使用方向键在表内定位，但仍然没办法指明以何种顺序定位。
  • 其实，没有哪种标记会告诉屏幕阅读器的用户，是应该通过行还是列阅读。有意义的标记仍然来自非表格语义内容。

响应表的方法

让我们来想一个不同的响应式的表方法：

  1. 压扁：如果列没有内容可以水平压扁，在手机屏幕上通常是没问题的，而且也没有改变布局，这是一个可行的方法。
  2. 水平滚动：如果布局和内容是准确的和关键的，用户可以向左向右滚动。这个可以用 CSS 的 overflow=”auto” 包装。
  3. 按行折叠：在小屏幕上把每行分割到一个小的单列表格中。把 display:table 换成 display:block 可以用普通表标记实现。
  4. 按列折叠：这可是个棘手的事情，因为代码的顺序是按行和用 标签包裹的，所以不能用纯 CSS 普通表标记完成。我们不得不要么改标记要么使用JavaScript实现。


这四种表格的视觉化例子

不推荐建立响应式表的方式

经过反复实验，我放弃了下面这些方法：

  1. 建立另一个窄的表格，通过 JavaScript 断点来隐藏、显示交替实现。
    为什么不呢？内容重复，也不比一个 “.m” 站点好，还会打散表内的特殊 ID。风格指南驱动组件不是一个好主意。

  2. 使用普通表标记和在 JavaScript 断点的处把表重新安排进一个响应式的里面。
    为什么不呢？需要对垂直和水平的表进行不同的标记。这会打破表内容的 JS 初始化，还需要大量的 JS 事件侦听器和 DOM 操作。

  3. 保持表格标记，但是切换到 display:flex 以垂直对其表内容。
    为什么不呢？不可能通过 标签跨行对其单元格，并且 display: table-cell 会覆盖 flex-item。

flexbox 的响应表

1a) 面向行型的表…

  • 使用语义标题和内容使手机或屏幕阅读器能够读取有序的标记。
  • 抛弃“行”包装的概念。
  • 为每个单元的宽度设置每个单元的百分比数字的宽度。自动调整列宽是不可能的。

给 HEXO 博客自定义背景图

发表于 2016-03-22   |   分类于 个人笔记   |  

最近我在 github 上搭建了个 Hexo 博客，安装步骤就不再多说了，基本都是按照网上的教程来的，在此要感谢：

  • 叶阳栩宁的教程
  • Zipperary 的教程
  • IIssNan 的 NexT 主题

虽然有现成的教程，但是在搭博客的时候还是走了点弯路，碰到些奇奇怪怪的问题，好在有 Google 大法，解决了不少问题。

博客搭建完之后，只有一篇自动生成的 Hello World，于是我把最近个人翻译的两篇关于 CSS 的文章也发了上来，于是博客便有了点自己的东西了：

NexT 主题有三种样式，都是这种白底黑字很淡雅的，但是我看着白色的背景挺单调的，就想着如何用刚学的 CSS 来给我的博客添加背景图片，略微“美化”一下。

在 NexT 主题的 github 的 issue 里，我找到了如何自定义一些页面样式的方法，作者 IISSNan 很贴心的在主题的 “\source\css_custom\” 文件夹下放了一个 custom.styl 文件，可以让用户自己在里面写 CSS 代码以个性化自己的博客。
于是我找到这个文件，并找到一张去年在三亚拍的大海照片用作背景，开始编写代码了！
首先我尝试输入了如下的 CSS 代码，主要是来测试一下这个 custom.styl 是否真的有效：

body{
background-image:url(/images/background.jpg);
}

保存之后，刷新一下本地博客的网页（毕竟还没有 deploy 部署上传），看来我已经成功地给博客加上背景图片了，显然这离美观还差好大一截呢，文章比较长的时候，图片就又重复了一张出来，如图：

因此，我想做到：

  1. 图片平铺在页面上
  2. 平铺之后不会因为往下滚动图片就会消失

于是添加定义 background-repeat 和 background-attachment 属性的代码：

body{
background-image:url(/images/background.jpg);
background-repeat: no-repeat;
background-attachment:fixed;
}

但是又出现了新的问题，背景图只有左上角的一部分区域平铺到网页之中，其他的都看不到了，我的照片只看得到蓝天，看不到大海了，查看 W3school 发现使用 background-position:50% 50%; 代码可以解决！

这会导致图像适当放置，其中心与其元素的中心对齐。换句话说，百分数值同时应用于元素和图像。也就是说，图像中描述为 50% 50% 的点（中心点）与元素中描述为 50% 50% 的点（中心点）对齐。

于是再次修改代码：

body{
background-image:url(/images/background.jpg);
background-repeat: no-repeat;
background-attachment:fixed;
background-position:50% 50%; 
}

现在，背景图的设置就基本告一段落了。再浏览一下，当我向下滚动条到最底的时候，发现页脚最下方的版权文字是浅灰色的，在之前 NexT 主题默认的纯白的背景色上没问题，然而在我的背景图上就会不那么明显。

于是我选择继续修改 CSS 代码，想个办法给这几个字改一下颜色吧。
通过 Chrome 浏览器自带的开发者工具，可以找到页脚所在的文字是在 footer 的类下面，通过右边修改 color 的颜色值可以所见即所得地调整出一个我想要的颜色。然后，我继续在 custom.styl 文件里添加如下的代码：

footer{
color:#eee;
}

刷新之后，再次发现字体颜色已经变成我想要的颜色了。

我的博客已经搭建完毕了，以后我会将学习的笔记和个人心得等放上来，也欢迎各位访问和交流。

译文：CSS多栏布局新手教程

发表于 2016-03-19   |   分类于 个人译文   |  

原文：
A CSS Multi-column Layout Tutorial for Beginners

对有的人来说，阅读一段有很长文字内容的时候可能会出现这样的问题，不是丢字了就是落行了。这个问题很容易通过把内容以多栏进行布局的方式解决。文字的多栏化在出版物中几乎无处不在。利用 CSS 多栏布局模块的功能可以使我们在网站上也能制作出类似的多栏效果。

有个问题是我们在设计网页时，是很难控制多栏大小的。在本次教程中，我会教你如何创建一个精美的且能够自适应各种屏幕尺寸的响应式多栏布局。我们先从最基本的开始，然后再深入到更复杂的概念。

浏览器支持

如果你在代码里加上一些前缀的话，浏览器就能够很好地支持多栏布局了。根据 Can I Use 的全球统计来看，95.32％浏览器都是支持多栏的。像 IE10 及以上，EDGE 和 Opera Mini 等浏览器能够完整支持多栏布局。其他的比如 Firefox 和 Chrome 浏览器需要加前缀代码。

如果你想兼容老版本浏览器（IE9 及以下）的话，有个 老的补丁 你可以试试。当然，如果一个浏览器根本不支持多栏效果的话，就会变回单栏布局。因此，使用补丁可能不是最好的选择。

CSS 多栏布局模块有许多不同的属性。下面我会一一的解释。

column-count 和 column-width

column-count 属性可以给一个元素设定一定数量的栏。你可以将其设置为 auto 或是一个整数。如果设置为 auto，栏数将根据 column-width 属性自动确定。如果设置为一个整数，所有的栏会被设置成相等的宽度。

column-width 属性可以给一个元素设定一个特定的栏宽。但并非严格如此的，如果没有足够的空间栏会变窄。该属性也可以同时设置一个 auto 或一个整数的值。如果设置为 auto ，宽度将被 column-count 属性确定，所有栏平均分配总宽度。

还有，这两个值可以同时使用 columns 属性来简写设置。columns 属性语法是：

.example { columns: <'column-width'> || <'column-count'> }

下面是这个属性的几个例子，每个旁边有相应的解释：

.example {
columns: 10em;  /* column-width: 10em / column-count: auto */
columns: 4; /* column-width: auto / column-count: 4 */
columns: 4 auto;/* column-width: auto / column-count: 4 */
columns: auto 10em; /* column-count: auto / column-width: 4(译者注：原文错，这里应该是10em) */
columns: auto;  /* column-count: auto / column-width: auto */
columns: auto auto; /* column-count: auto / column-width: auto */
}

你可以看到，第一个 columns 的简写和第四个的定义是一样的，而第二个和第三个也是一样的。也就是说，如果一个整数没有单位的话，就会被解析为 column-count。

可以到 CodePen 看一下上述的 Demo。

如果调整一下窗口大小，你会发现：

  • column-count 属性始终保持你设置的栏数，只有栏宽会变。
  • column-width 属性会根据可用空间自动变化。这种方式调整的栏宽会比规定栏数来的更好。如果没有足够的可用空间，它也可以调整栏宽变的很小。
  • columns 属性使用 column-count 值限制最大的栏。这样的方式调节宽度 column-count 不会超出限制并且 column-width 也会更接近设定的宽度。

    column-gap 和 column-rule

    column-gap 属性允许我们设定栏和栏之间的间隙。你可以将其设置为 normal 或是一个长度值，可以是零，但不能为负。当你将它设置为 normal，是会根据浏览器的默认值设置间隙。

column-rule 属性是一个简写，使我们能够添加栏之间的线。这个类似于 border-left 或 border-right 属性。此属性的语法如下：

.example {
column-rule: <'column-rule-width'> || <'column-rule-style'> || <'column-rule-color'>
}

对于 column-rule-width，你可以像指定 3 px 这样一个宽度或使用关键字：thin，medium 或 thick。column-rule-style 可以使用 dashed，dotted，solid 等值，你可以使用 border-style 的有效值定义 column-rule-style，并且 column-rule-color 可以使用任何有效的颜色值。

可以到 CodePen 看一下上述的 Demo。

column-fill 和 column-span

column-fill 属性可以确定文字是如何被分配到各栏当中的。这个属性可以设置为 auto 或 balance。如果设置为 auto，依次填充进每栏。采用 balance 可以确保文字会被所有栏平分。

有一点要提醒的是，如果你一个栏设置了固定的高度，Firefox 会自动平分文字内容，而其他的浏览器会依次填充进每栏。

column-span 属性可以控制多个栏如何跨越。它有两个值：all 或 none。当设置为 all 的元素在所有栏跨越。此属性 Firefox 不支持。

可以到 CodePen 看一下上述的 Demo。

在 Firefox 中，blockquote 会位于中间栏的位置，这也是可接受的。

创建具有响应式的多栏布局

现在，你已经迅速掌握了以上这几种新属性和它们的值，我们再来专注来做好响应式布局以促进用户体验。

column-count 和 column-width 都有一些的问题。当在大屏设备中 column-count 保持在相应的栏数，但是在小屏设备上布局在会被打破。同样，如果 column-width 在小屏设备上栏虽然不会太窄，但在大屏设备上导致会有大量的栏出现。

为了确保您的布局在大小屏幕上都能较好的适应，你应该同时指定 column-count 和 column-width 参数。这样你就能控制好栏的宽度和数量。还有一些小问题，我们下一步继续来解决。

固定水平滚动

如果您指定一个固定高度的栏，如果使得窗口较窄则会出现一个水平滚动条。由于内容不能垂直展开，就会水平地增长。为了解决这个问题，你可以调整你的浏览器的宽度时，水平滚动条首先出现。然后，使用媒介查询属性在对该栏的高度设置一个 auto 的属性值。下面是代码：

.responsive-height {
height: 250px;
}
@media (max-width: 1040px) {
.responsive-height {
height: auto;
}
}

可以到 CodePen 看一下上述的 Demo。

调整窗口的大小，看看这两个例子是怎么响应的。

固定很长的栏

如果你的栏太长，用户需要向上和向下滚动才能阅​​读所有内容，这太麻烦了。当栏的高度比窗口高度还大，不用多栏是一个好方法。这可以再次使用媒介查询属性来解决：

@media (min-width: 800px) {
.long-columns {
columns: 3 200px;
}
}

在只有宽度足够的情况下，才会有多栏，用户再也不用上下滚动。

可以到 CodePen 看一下上述的 Demo。

结论

我希望这篇 CSS 中多栏布局模块的介绍能让你熟悉这个功能。这个属性可以提高你的响应式设计能力，如果你仍然想要兼容旧的浏览器，多栏还是会变成单个栏的。
