Getting Started with CSS Modules

没有一个单一的方法，只用 CSS 模块就能使得 JavaScript 模板，CSS 文件或编译器工作。在这篇 CSS 模块系列文章的第二篇中，我们将学习一个方法。本文章的目的旨在建立一个 CSS 模块的项目并使之成功启动和运行。

第 2 部分：如何开始使用 CSS 模块（就是现在这篇！）

在我工作的项目，有个 CSS 不依赖客户端 JavaScript 就能生效的要求，因此编译器需要在 HTML 和 CSS 部署之前就把一切都运作好。我们将使用 Webpack，一种编译系统或模块打捆器。在接下来的文章中我们将重点放在使下面静态的 HTML 代码呈现到浏览器的这么一个真实的例子。

现在开始！

安装 WebPack

在安装完 NPM 和 Node.js 之后，新建一个空目录，并运行以下命令：

npm init --y

这会生成一个package.json文件，并自动填入一些默认设置。这是一个附件清单——当人们下载和用npm install命令安装这个项目的时候，需要有哪些文件的说明。

Webpack会管理这个编译过程。它会密切关注我们的 CSS，JavaScript 和 HTML 互相执行的所有关联。但 Webpack 到底是什么呢？Maxime Fabre 认为它是一种编译系统或模块打捆器：

是的，它将两者结合了。Webpack 不生成资源，认为资源是模块本身，然后分别打捆进模块中。它可以被导入、修改、操作并且最后可以打包进最终的的包。

如果这听起来不可思议，别担心。想一下 Sass 和 Glup 还有 npm 都还陌生和困难吗？我们会搞定的。

首先确保 Webpack “捆绑”模块正常并定义一个依赖的 JavaScript 文件，它可以导入代码。我们需要在全局下安装 Webpack，在终端中输入命令：

npm install webpack -g

完成后，我们在我们本地项目中启用 Webpack，代码如下：

npm i -D webpack

我们在/src目录下生成一个index.js文件。通常我喜欢给静态资源建立文件夹（比如图像、字体、CSS 文件和标记）。我写的代码在/src目录下，机器自动生成的在/build下。我的想法是删除/build目录也是可以的，并不会有任何问题，因为我们可以运行一个命令只处理/src目录里的东西，并重建/build目录。

在/src目录里，我们可以新建一个空alert.js文件（下面一会儿还会提到）。我们还需要在根目录新建一个webpack.config.js文件，/src目录应该是这样构成的：

package.json
webpack.config.js
/node_modules
/src
  index.js
  alert.js

在webpack.config.js（Webpack 的设置文件）文件中，我们添加如下代码：

module.exports = {
  entry: './src',
  output: {
    path: 'build',
    filename: 'bundle.js',
  },
};

当我们在这运行webpack命令时，Webpack 将/src中所有的资源文件建立一个文件关系树。

回到src/index.js文件，我们可以添加代码：

require("./alert.js");

在alert.js文件里，我们可以这样写：

alert("LOUD NOISES");

然后我们在根目录下建一个index.html文件，并在<body>闭合前添加<script>标签并添加打捆脚本：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document name</title>
</head>
<body>
    <h1>CSS Modules demo</h1>

    <script src="build/bundle.js"></script>
</body>
</html>

Webpack 会生成bundle.js文件，使用webpack命令即可。为了更便于使用，我们可以更新package.json。下面是文件中代码：

"scripts": {
  "test": "echo 'Error: no test specified' && exit 1"
},

这是npm给的默认代码，我们可以用下面的代码替换掉它，它可以执行 Webpack 命令并打开一个浏览器窗口：

"scripts": {
  "start": "webpack && open index.html"
},

快看，有效果了！这证明了我们的index.js文件导入了alert.js的代码，Webpack 正确地捆绑了一切。如果我们现在删除alert.js文件，我们再次运行npm start的时候会报错：

添加第一个加载器

在 Webpack 中加载程序是非常重要的。Maxime Fabre 在这篇文章里就提到过：

加载器是个小插件，基本上是：“当你遇到这种文件时，就执行它”。

在 Maxime 的教程他提到了 Babel 加载器，这是一个好的开始，Babel 允许我们使用 ES2015 和 JavaScript 语言的最新改进。所以不需要Common.js函数的话需要提前require另一个模块，现在我们可以用import代替了。用 Babel 我们可以用类、方向函数等这些很酷的功能：

像 Babel 这样的工具可以让我们能够编写新的 ES2015 的代码，并执行一项称为 transpiling 的任务(类似预处理)用以把代码转换到以前版本的 JavaScript 中以提高浏览器兼容性。这个类似于 Sass，Sass 语法中编写的代码，通过预处理器编译成标准的 CSS。

如果想要运行 Babel 的话，我们需要安装 Webpack Babel 加载器：

npm i -D babel-loader babel-core babel-preset-es2015

在项目的根目录中的.babelrc文件里，我们可以配置默认值用以让别人知道用什么 JavaScript 语法：

{ "presets": ["es2015"] }

我们希望只运行我们自己写的.js文件，我们之后安装生成的文件有自己的语法，我们不希望该代码受影响。这就要靠 Webpac 加载器来发挥作用。我们可以打开webpack.config.js文件，并改写成代码：

module.exports = {
  entry:  './src',
  output: {
  path: 'build',
    filename: 'bundle.js',    
  },
  module: {
    loaders: [
      {
        test: /\.js/,
        loader: 'babel',
        include: __dirname + '/src',
       }
    ],
  }
};

loders下面的test键值对告诉 Webpack 哪种类型的文件需要执行，include 准确地告诉了需要执行操作的项目路径。

让我们来测试一下 Babel 与 Webpack 的协同工作。在一个新的文件（src/robot.js），这样写：

const greetings = (text, person) => {
  return `${text}, ${person}. I read you but I’m sorry, I’m afraid I can’t do that.`;
}

export default greetings;

这个 JavaScript 文件是用了一些 ES2015 的特殊功能，比如 export，const和let，还有箭头功能和模板文字。

我们可以这样导入模块到src/index.js文件中：

import greetings from './robot.js'
document.write(greetings("Affirmative", "Dave"));

最后，我们需要做的就是再次运行npm start，浏览器应该弹出返回文字：

“Affirmative, Dave. I read you but I’m sorry, I’m afraid I can’t do that.”

很显然，Babel 生效了。

这不是一个 CSS 模块然而很接近了。在我们继续下一步之前，先删除src/robot.js和src/index.js里的所有代码。

载入样式表

现在，我们需要让所有模板几乎都能有效，我们需要增加两个加载器：css-loader和style-loader，我们先安装：

npm i -D css-loader style-loader

css-loader 程序需要一个 CSS 文件并读取其所有关联项，而 style-loader 将这些样式直接嵌入到标记中。让我们在src/app.css中编写一些 CSS 测试一下：

.element {
  background-color: blue;
  color: white;
  font-size: 20px;
  padding: 20px;
}

我们可以import样式表到src/index.js文件中：
import styles from ‘./app.css’

let element = `
  <div class="element">
    <p>Lorem ipsum dolor sit amet, consectetur adipisicing elit. Consequatur laudantium recusandae itaque libero velit minus ex reiciendis veniam. Eligendi modi sint delectus beatae nemo provident ratione maiores, voluptatibus a tempore!</p>
  </div>
`

document.write(element);

哇哦！我们只是使样式表成为一个 JavaScript 文件的关联项吗？是的，我们做到了。但在此之前的正常生效之前，我们需要再次重新配置webpack.config.js：

module.exports = {
  entry:  './src',
  output: {
    path: 'build',
      filename: 'bundle.js',
    },
  module: {
    loaders: [
      {
        test: /\.js/,
    loader: 'babel',
    include: __dirname + '/src',
  },
  {
    test: /\.css/,
    loaders: ['style', 'css'],
    include: __dirname + '/src'
      }
    ],
  }
};

运行npm start之后就会变成这样：

因此，如果我们对我们的文档“检查元素”，我们会发现 style-loader 已经把该文件插入进<style>标记里的<head>之中：

我们总结一下刚才发生了什么。我们生成了一个 JavaScript 文件，它请求另一个 CSS 文件，然后把代码嵌入在网页里。举一个更实际点的例子，我们创建一个buttons.js文件，并和buttons.css关联起来，然后导入 JavaScript 到另一个文件，再组织模板和生成一些 HTML 文件。这使我们的代码模块化，易于阅读。

就个人而言，为了保持代码清爽，我宁愿用单独的 CSS 文件，而不是将所有的代码写在一起。要这么做的话，我们需要使用Webpack插件——extract text：

把每个进入块的请求（style.css）输出到一个单独的 CSS 文件。因此，样式表不再联到的 JavaScript，而是在单独的 CSS 包文件（styles.css）中。如果你的总样式表很大，样式表会并行加载到 JavaScript 中，因此会让这变得快一些。

我们必须通过 NPM 安装插件 ：

npm i -D extract-text-webpack-plugin

我们再次通过请求更新webpack.config.js文件，并加载 CSS 到其中：

e.exports = {
  entry:  './src',
  output: {
    path: 'build',
    filename: 'bundle.js',
  },
  module: {
loaders: [
  {
    test: /\.js/,
    loader: 'babel',
    include: __dirname + '/src',
  },
  {
    test: /\.css/,
    loader: ExtractTextPlugin.extract("css")
  }
],
  },
  plugins: [
new ExtractTextPlugin("styles.css")

Extract Text 插件为我们生成了一个styles.css文件!

你可能已经注意到我们已经完全摆脱了 style-loader，因为我们再也不用把样式注入到标记中了。所以如果我们打开/build目录，会发现里面生成了一个已经写好代码的styles.css文件。通过index.html文件，我们把样式加入到<head>标签里：

<link rel="stylesheet" href="build/styles.css">

再次运行npm start，看！我们的样式代码神奇地出现在页面上原本就属于他们的地方。

现在，在有 CSS 和 HTML 的页面上，我们如何操纵类名，以获得本地作用域的优点呢？我们要做的就是像这样更新 webpack.config.js文件：

/\.css/,
  loader: ExtractTextPlugin.extract('css?modules&importLoaders=1&localIdentName=[name]__[local]___[hash:base64:5]'),

这会在该类文件名的末尾生成一些奇怪的文本。这是 CSS 模块的哈希值，通过 Webpack 中 CSS 加载器来给类添加的。
下面，我们通过styles.element类来更新index.js文件：

 styles from './app.css'
let element = `
  <div class="${styles.element}">
<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit. Consequatur laudantium recusandae itaque libero velit minus ex reiciendis veniam. Eligendi modi sint delectus beatae nemo provident ratione maiores, voluptatibus a tempore!</p>
  </div>
`
document.write(element);

看看发生了什么，再次运行npm start之后，我们的代码经过 Webpack 处理，让本地域不再是一个难题。因为类已经像这样地注入到网页之中了：

<div class="app__element___1MmQg">

我们还没有真正完成，因为有许多问题没有答案。我们怎么能更进步地写代码？我们如何避免在网页的标记中写那麻烦的 document.write 语句？我们应如何组织我们的模块和文件？现在 CSS 模块我们只用到一半，下面我们要考虑如何从另外的系统导入代码库。

在接下来的教程中，我们会看到 React 是如何帮助我们生成整洁的小模块，也会学习到如何在很多模板里生成静态标签，以及如何给我们的项目添加 Sass 和 PostCSS 这样的其他功能。
