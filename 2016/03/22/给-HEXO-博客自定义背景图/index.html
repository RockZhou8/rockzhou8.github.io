给 HEXO 博客自定义背景图

发表于 2016-03-22   |   分类于 个人笔记   |  

最近我在 github 上搭建了个 Hexo 博客，安装步骤就不再多说了，基本都是按照网上的教程来的，在此要感谢：

  • 叶阳栩宁的教程
  • Zipperary 的教程
  • IIssNan 的 NexT 主题

虽然有现成的教程，但是在搭博客的时候还是走了点弯路，碰到些奇奇怪怪的问题，好在有 Google 大法，解决了不少问题。

博客搭建完之后，只有一篇自动生成的 Hello World，于是我把最近个人翻译的两篇关于 CSS 的文章也发了上来，于是博客便有了点自己的东西了：

NexT 主题有三种样式，都是这种白底黑字很淡雅的，但是我看着白色的背景挺单调的，就想着如何用刚学的 CSS 来给我的博客添加背景图片，略微“美化”一下。

在 NexT 主题的 github 的 issue 里，我找到了如何自定义一些页面样式的方法，作者 IISSNan 很贴心的在主题的 “\source\css_custom\” 文件夹下放了一个 custom.styl 文件，可以让用户自己在里面写 CSS 代码以个性化自己的博客。
于是我找到这个文件，并找到一张去年在三亚拍的大海照片用作背景，开始编写代码了！
首先我尝试输入了如下的 CSS 代码，主要是来测试一下这个 custom.styl 是否真的有效：

body{
background-image:url(/images/background.jpg);
}

保存之后，刷新一下本地博客的网页（毕竟还没有 deploy 部署上传），看来我已经成功地给博客加上背景图片了，显然这离美观还差好大一截呢，文章比较长的时候，图片就又重复了一张出来，如图：

因此，我想做到：

  1. 图片平铺在页面上
  2. 平铺之后不会因为往下滚动图片就会消失

于是添加定义 background-repeat 和 background-attachment 属性的代码：

body{
background-image:url(/images/background.jpg);
background-repeat: no-repeat;
background-attachment:fixed;
}

但是又出现了新的问题，背景图只有左上角的一部分区域平铺到网页之中，其他的都看不到了，我的照片只看得到蓝天，看不到大海了，查看 W3school 发现使用 background-position:50% 50%; 代码可以解决！

这会导致图像适当放置，其中心与其元素的中心对齐。换句话说，百分数值同时应用于元素和图像。也就是说，图像中描述为 50% 50% 的点（中心点）与元素中描述为 50% 50% 的点（中心点）对齐。

于是再次修改代码：

body{
background-image:url(/images/background.jpg);
background-repeat: no-repeat;
background-attachment:fixed;
background-position:50% 50%; 
}

现在，背景图的设置就基本告一段落了。再浏览一下，当我向下滚动条到最底的时候，发现页脚最下方的版权文字是浅灰色的，在之前 NexT 主题默认的纯白的背景色上没问题，然而在我的背景图上就会不那么明显。

于是我选择继续修改 CSS 代码，想个办法给这几个字改一下颜色吧。
通过 Chrome 浏览器自带的开发者工具，可以找到页脚所在的文字是在 footer 的类下面，通过右边修改 color 的颜色值可以所见即所得地调整出一个我想要的颜色。然后，我继续在 custom.styl 文件里添加如下的代码：

footer{
color:#eee;
}

刷新之后，再次发现字体颜色已经变成我想要的颜色了。

我的博客已经搭建完毕了，以后我会将学习的笔记和个人心得等放上来，也欢迎各位访问和交流。
