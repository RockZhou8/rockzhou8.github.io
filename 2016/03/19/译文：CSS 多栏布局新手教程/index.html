原文：
A CSS Multi-column Layout Tutorial for Beginners

对有的人来说，阅读一段有很长文字内容的时候可能会出现这样的问题，不是丢字了就是落行了。这个问题很容易通过把内容以多栏进行布局的方式解决。文字的多栏化在出版物中几乎无处不在。利用 CSS 多栏布局模块的功能可以使我们在网站上也能制作出类似的多栏效果。

有个问题是我们在设计网页时，是很难控制多栏大小的。在本次教程中，我会教你如何创建一个精美的且能够自适应各种屏幕尺寸的响应式多栏布局。我们先从最基本的开始，然后再深入到更复杂的概念。

浏览器支持

如果你在代码里加上一些前缀的话，浏览器就能够很好地支持多栏布局了。根据 Can I Use 的全球统计来看，95.32％浏览器都是支持多栏的。像 IE10 及以上，EDGE 和 Opera Mini 等浏览器能够完整支持多栏布局。其他的比如 Firefox 和 Chrome 浏览器需要加前缀代码。

如果你想兼容老版本浏览器（IE9 及以下）的话，有个 老的补丁 你可以试试。当然，如果一个浏览器根本不支持多栏效果的话，就会变回单栏布局。因此，使用补丁可能不是最好的选择。

CSS 多栏布局模块有许多不同的属性。下面我会一一的解释。

column-count 和 column-width

column-count 属性可以给一个元素设定一定数量的栏。你可以将其设置为 auto 或是一个整数。如果设置为 auto，栏数将根据 column-width 属性自动确定。如果设置为一个整数，所有的栏会被设置成相等的宽度。

column-width 属性可以给一个元素设定一个特定的栏宽。但并非严格如此的，如果没有足够的空间栏会变窄。该属性也可以同时设置一个 auto 或一个整数的值。如果设置为 auto ，宽度将被 column-count 属性确定，所有栏平均分配总宽度。

还有，这两个值可以同时使用 columns 属性来简写设置。columns 属性语法是：

.example { columns: <'column-width'> || <'column-count'> }

下面是这个属性的几个例子，每个旁边有相应的解释：

.example {
columns: 10em;  /* column-width: 10em / column-count: auto */
columns: 4; /* column-width: auto / column-count: 4 */
columns: 4 auto;/* column-width: auto / column-count: 4 */
columns: auto 10em; /* column-count: auto / column-width: 4(译者注：原文错，这里应该是10em) */
columns: auto;  /* column-count: auto / column-width: auto */
columns: auto auto; /* column-count: auto / column-width: auto */
}

你可以看到，第一个 columns 的简写和第四个的定义是一样的，而第二个和第三个也是一样的。也就是说，如果一个整数没有单位的话，就会被解析为 column-count。

可以到 CodePen 看一下上述的 Demo。

如果调整一下窗口大小，你会发现：

  • column-count 属性始终保持你设置的栏数，只有栏宽会变。
  • column-width 属性会根据可用空间自动变化。这种方式调整的栏宽会比规定栏数来的更好。如果没有足够的可用空间，它也可以调整栏宽变的很小。
  • columns 属性使用 column-count 值限制最大的栏。这样的方式调节宽度 column-count 不会超出限制并且 column-width 也会更接近设定的宽度。

    column-gap 和 column-rule

    column-gap 属性允许我们设定栏和栏之间的间隙。你可以将其设置为 normal 或是一个长度值，可以是零，但不能为负。当你将它设置为 normal，是会根据浏览器的默认值设置间隙。

column-rule 属性是一个简写，使我们能够添加栏之间的线。这个类似于 border-left 或 border-right 属性。此属性的语法如下：

.example {
column-rule: <'column-rule-width'> || <'column-rule-style'> || <'column-rule-color'>
}

对于 column-rule-width，你可以像指定 3 px 这样一个宽度或使用关键字：thin，medium 或 thick。column-rule-style 可以使用 dashed，dotted，solid 等值，你可以使用 border-style 的有效值定义 column-rule-style，并且 column-rule-color 可以使用任何有效的颜色值。

可以到 CodePen 看一下上述的 Demo。

column-fill 和 column-span

column-fill 属性可以确定文字是如何被分配到各栏当中的。这个属性可以设置为 auto 或 balance。如果设置为 auto，依次填充进每栏。采用 balance 可以确保文字会被所有栏平分。

有一点要提醒的是，如果你一个栏设置了固定的高度，Firefox 会自动平分文字内容，而其他的浏览器会依次填充进每栏。

column-span 属性可以控制多个栏如何跨越。它有两个值：all 或 none。当设置为 all 的元素在所有栏跨越。此属性 Firefox 不支持。

在 Firefox 中，blockquote 会位于中间栏的位置，这也是可接受的。

创建具有响应式的多栏布局

现在，你已经迅速掌握了以上这几种新属性和它们的值，我们再来专注来做好响应式布局以促进用户体验。

column-count 和 column-width 都有一些的问题。当在大屏设备中 column-count 保持在相应的栏数，但是在小屏设备上布局在会被打破。同样，如果 column-width 在小屏设备上栏虽然不会太窄，但在大屏设备上导致会有大量的栏出现。

为了确保您的布局在大小屏幕上都能较好的适应，你应该同时指定 column-count 和 column-width 参数。这样你就能控制好栏的宽度和数量。还有一些小问题，我们下一步继续来解决。

固定水平滚动

如果您指定一个固定高度的栏，如果使得窗口较窄则会出现一个水平滚动条。由于内容不能垂直展开，就会水平地增长。为了解决这个问题，你可以调整你的浏览器的宽度时，水平滚动条首先出现。然后，使用媒介查询属性在对该栏的高度设置一个 auto 的属性值。下面是代码：

.responsive-height {
height: 250px;
}
@media (max-width: 1040px) {
.responsive-height {
height: auto;
}
}

可以到 CodePen 看一下上述的 Demo。

调整窗口的大小，看看这两个例子是怎么响应的。

固定很长的栏

如果你的栏太长，用户需要向上和向下滚动才能阅​​读所有内容，这太麻烦了。当栏的高度比窗口高度还大，不用多栏是一个好方法。这可以再次使用媒介查询属性来解决：

@media (min-width: 800px) {
.long-columns {
columns: 3 200px;
}
}

在只有宽度足够的情况下，才会有多栏，用户再也不用上下滚动。

结论

我希望这篇 CSS 中多栏布局模块的介绍能让你熟悉这个功能。这个属性可以提高你的响应式设计能力，如果你仍然想要兼容旧的浏览器，多栏还是会变成单个栏的。
